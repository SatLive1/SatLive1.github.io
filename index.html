SatLive

Toward End-to-End Live Video Streaming over Integrated Satellite-Terrestrial Networks

WWW'26, April 13-17, 2026, Dubai, UAE

About SatLive

SatLive is a high-performance framework designed for end-to-end live video streaming over integrated satellite-terrestrial networks (ISTNs). It addresses the core challenges of ISTNs—high satellite mobility, network dynamics, and QoE requirements—through a hierarchical architecture and intelligent optimization mechanisms.

By leveraging medium earth orbit (MEO) satellites to manage low earth orbit (LEO) constellations, SatLive achieves adaptive routing and efficient data transmission, enabling seamless video streaming even in remote areas or during terrestrial network disruptions.

Key Features

Hierarchical Architecture

MEO satellites act as regional controllers for LEO constellations, providing global perspective for routing decisions while maintaining low control overhead.

QoE-Aware Routing

Lightweight federated learning model (MARL) optimizes paths based on latency, packet loss, and video-specific constraints (bitrate, rebuffering).

Video-Aware Transmission

Frame prioritization (I/P/B frames) and time-interleaved redundancy ensure key video data is protected against network fluctuations.

Dynamic Adaptability

Real-time path repair and cross-domain handover mechanisms adapt to LEO satellite mobility and link state changes.

High Efficiency

Reduces average packet delay by at least XX%, improves bitrate by XX%, and cuts rebuffering time by XX% compared to state-of-the-art solutions.

Global Coverage

Supports streaming in unserved/underserved areas (remote regions, disaster zones) through LEO-MEO-Ground integrated networks.

System Architecture

SatLive adopts a three-layer management architecture to handle ISTN dynamics and video streaming requirements:

1. MEO Satellite Layer

Responsible for routing strategy generation and optimization. Runs a multi-agent reinforcement learning (MARL) model under the centralized training, decentralized execution (CTDE) principle.

2. LEO Satellite Layer

Senses link states, communicates with ground stations (GSes), and forwards data packets. Each LEO is anchored to a managing MEO satellite.

3. Ground Station (GS) Layer

Entry point for video streams. Selects optimal LEO access satellites based on SNR, RTT, and link duration to ensure transmission stability.

Experimental Results

Evaluated on StarPerf simulator with real Starlink data, SatLive outperforms baselines (Dijkstra, CMADR, StarStream) in key metrics:

Metric SatLive Performance Advantage Over Baselines
End-to-End Latency (9 pps) ~570 ms 30-40% lower than Dijkstra/CMADR
Packet Loss Rate (9 pps) ~1% 3-6x lower than Dijkstra/CMADR
Delivery Rate (9 pps) ~80% 10-30% higher than Dijkstra/CMADR
QoE Score 0.84 7-12% higher than StarStream/Robust schemes
Total Stall Time Average 5.3 s Minimal stalling (90% of experiments have 0 stalls)

Getting Started

Prerequisites

  • StarPerf Simulator (for network environment emulation)
  • FFmpeg Libraries (for video encoding/decoding)
  • Python 3.8+ (for MARL model and framework execution)
  • PyTorch/TensorFlow (for reinforcement learning training)

Installation

Clone the repository and install dependencies:

git clone https://SatLive.github.io.git
cd SatLive
pip install -r requirements.txt

Configuration

Refer to the config.yaml file to adjust network parameters (satellite orbit, link characteristics) and video settings (resolution, bitrate, GOP size).

Deployment

Run the main execution script to start the framework:

python satlive_main.py --config config.yaml
